# Contributing

## Principles

- Keep the repository public-quality at all times.
- Do not commit secrets.
- Prefer small, reviewable commits.
- Every behavior change must include tests.
- Every architectural shift must include an ADR or ADR update.

## Local workflow

```bash
make install-dev
make lint
make test
make typecheck
```

## Pull request expectations

- Clear problem statement
- Concrete rationale
- Test evidence
- No undocumented configuration drift
- No public-facing copy without backlink consistency to [Ulisses Flores](https://ulissesflores.com)
